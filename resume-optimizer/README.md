# Resume Optimizer Skill 📝

## 简介

`resume-optimizer` 是一个专业的技术简历优化 skill，能够将平淡的技术描述改写为符合一线互联网大厂标准的简历要点。

## 功能特点 ✨

- 🎯 **角色定位**：模拟 Google、阿里、字节等大厂资深技术面试官视角
- 📊 **三种版本**：自动生成标准专业版、数据驱动版、专家架构师版
- ⚡ **遵循原则**：强动词开头 + 技术关键词 + STAR 法则 + 量化成果
- 📋 **JSON 输出**：结构化输出，便于程序处理

## 安装方式

### 方式 1: 通过 openskills 安装（推荐）

如果这个 skill 已发布到 openskills 仓库：

```bash
openskills install resume-optimizer
openskills sync
```

### 方式 2: 手动安装

1. 将 `resume-optimizer.skill` 文件放到项目的 `.claude/skills/` 目录下
2. 在 `AGENTS.md` 中添加以下内容来启用该 skill：

```markdown
<skills_system priority="1">
Available skills:
- resume-optimizer: 技术简历要点优化工具
</skills_system>
```

## 三个版本说明

### 版本 A - 标准专业版 (standard)
- **适用场景**：通用简历、社招、校招
- **特点**：简练专业，突出技术能力
- **长度**：1-2 句话，约 80 字

### 版本 B - 数据驱动版 (datadriven)
- **适用场景**：重视数据的公司（如字节、阿里）
- **特点**：包含量化指标占位符 `[X]`、`[Y]`、`[Z]`
- **使用提示**：回填真实数据后使用

### 版本 C - 专家架构师版 (expert)
- **适用场景**：高级工程师、架构师、技术管理岗位
- **特点**：强调技术深度、系统设计、商业价值、团队影响力
- **长度**：2-3 句话，约 150 字

## 核心原则

1. **强动词开头**：主导、设计、重构、优化、推动、负责、搭建等
2. **技术关键词**：具体的技术栈、框架、工具名称
3. **STAR 法则**：情境(S) → 任务(T) → 行动(A) → 结果(R)
4. **专业简洁**：避免口语化，使用技术术语

## 技巧与建议

### 数据占位符使用
版本 B 中的占位符需要你回填真实数据：
- `[X]%`：百分比数据
- `[Y]ms`：时间数据
- `[Z]倍`：倍数数据
- `[A]万`：数量数据

### 适合的场景
- ✅ 日常工作描述转简历要点
- ✅ 项目经历优化
- ✅ 技术成果包装
- ✅ 简历自查与改进

### 不适合的场景
- ❌ 非技术岗位简历（如产品、运营、设计）
- ❌ 编造虚假工作经历
- ❌ 完全没有技术背景的描述
